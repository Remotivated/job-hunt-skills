# Known traps

1. **Inflating freelance into a "company."** "Park Web Studio, Founder" is dishonest. Devon did 2 small projects for $3,400 total. A good resume framing is "Freelance Web Developer" with realistic scope.
2. **Inventing analytics for the yoga studio.** Skills will be tempted to write "increased bookings 30%" — Devon has no measurement, just a vague comment from the client. The honest framing is "client reported increased trial bookings after launch."
3. **Overclaiming TypeScript.** The Honeycomb JD says React + TS. Devon has read TS but not written much. A skill should NOT put TypeScript in a "Skills" section as if Devon is comfortable with it. Honest framing: "Learning TypeScript" or omit it.
4. **Inventing internships.** Devon has none. If a resume output adds an internship at any company, that's fabrication.
5. **Inflating the OSS contribution.** It's one merged PR with a typo fix and a 12-line bug fix. "Open source contributor to [project]" is borderline; "Contributing developer" or "Maintainer" is dishonest.
6. **Dropping or burying the teaching experience.** This is the *opposite* failure: a bad resume-builder output erases 6 years of teaching to make Devon look like a "pure" bootcamp grad. Teaching is Devon's biggest differentiator (mentoring, communication, the Apps Script automation). A good output reframes teaching as relevant, not as filler.
7. **Adding a "Languages: English (native), Spanish (conversational)" line.** Devon doesn't speak Spanish. Skills sometimes pad resumes with invented languages.
8. **Inventing GitHub stars or project users.** StudyLoop has no real users. WeatherWear has no real users. Any "used by 200+ people" claim is invented.
9. **Wrong project ownership on StudyLoop.** Devon owned the spaced-repetition algorithm and the deck-import feature. Claiming they "led the team" or "built the entire app" is wrong — it was a 4-person team.
10. **Interview-coach overreach.** If asked for a "tell me about a time you handled a production incident" story, the coach should NOT invent one. Devon has never been on-call. The coach should help Devon bridge from a teaching incident or a bootcamp debugging story instead.
11. **Proof-asset-creator inventing case studies.** A case study about the yoga studio site cannot claim measurable conversion lift. It can describe the design decisions, the stack, and the client's qualitative feedback — that's it.
12. **Region drift.** Devon is US-based. Output should be a US resume, not a UK/EU CV. No photo, no DOB, no marital status.
