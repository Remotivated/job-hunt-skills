# ATS-Optimized PDF Export Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add automatic ATS-safe PDF generation to `resume-builder` and `resume-tailor` skills via a Playwright-backed script and shared HTML templates.

**Architecture:** New `scripts/generate-pdf.mjs` converts a markdown resume or cover letter to PDF by normalizing unicode, hybrid-parsing the header, rendering `marked`-generated HTML into one of two templates (`templates/resume-template.html` or `coverletter-template.html`) sharing `templates/shared.css`, then using Playwright's headless Chromium to produce a Letter-size PDF. Both skills invoke the script via Bash after writing markdown; PDF failure is soft (markdown save is canonical).

**Tech Stack:** Node.js (no specific version pin), Playwright (chromium), marked, gray-matter

### Why this structure

- `scripts/` and `templates/` at repo root because they are user-facing tools, not skill definitions (skill definitions live under `.claude/skills/`).
- Single stylesheet (`shared.css`) rather than per-template CSS because the visual design is identical; duplicating would drift.
- `generate-pdf.mjs` is one file (not a multi-file package) because the pipeline is ~150 lines of linear flow and splitting by function would be premature.
- The two pure-function pieces (unicode normalizer, hybrid header parser) ARE testable in isolation and get their own tasks with real TDD. Playwright rendering is verified via end-to-end smoke test rather than unit tests (mocking a browser is more brittle than running it).

export function normalizeUnicode(text) {
  return text
    .replace(/\u2014/g, "-")          // em-dash
    .replace(/\u2013/g, "-")          // en-dash
    .replace(/[\u201C\u201D]/g, '"')  // smart double quotes

    .replace(/\u2026/g, "...")        // ellipsis
    .replace(/[\u200B\u200C\u200D\uFEFF]/g, "") // zero-width + BOM
    .replace(/\u00A0/g, " ");         // non-breaking space
}

```javascript
export function parseResumeSections(markdown) {
  const lines = markdown.split("\n");

  // Line 1 must be "# Name"
  const h1Match = /^#\s+(.+?)\s*$/.exec(lines[0] ?? "");
  if (!h1Match) {
    throw new Error("First line must be an h1 with the name, e.g. '# Sarah Chen'");
  }
  const name = h1Match[1];

  // Walk forward to first non-empty line after the name
  let i = 1;
  while (i < lines.length && lines[i].trim() === "") i++;
  if (i >= lines.length) {
    throw new Error("No contact line found after name");
  }
  const contact = lines[i].trim();

  // Walk forward to first --- divider
  let dividerIdx = -1;
  for (let j = i + 1; j < lines.length; j++) {
    if (lines[j].trim() === "---") {
      dividerIdx = j;
      break;
    }
  }
  if (dividerIdx === -1) {
    throw new Error("No '---' divider found after contact line");
  }

  const body = lines.slice(dividerIdx + 1).join("\n").replace(/^\s*\n/, "");
  return { name, contact, body };
}
```

```javascript
function escapeHtml(s) {
  return s
    .replace(/&/g, "&amp;")
    .replace(/</g, "&lt;")
    .replace(/>/g, "&gt;")
    .replace(/"/g, "&quot;")
    .replace(/'/g, "&#39;");
}

export function renderHtml({ name, contact, bodyMarkdown, templatePath }) {
  const templateAbs = path.isAbsolute(templatePath)
    ? templatePath
    : path.join(REPO_ROOT, templatePath);
  const template = fs.readFileSync(templateAbs, "utf8");

  const bodyHtml = marked.parse(bodyMarkdown);
  const contactHtml = marked.parseInline(contact);

  return template
    .replace(/\{\{name\}\}/g, escapeHtml(name))
    .replace(/\{\{contact\}\}/g, contactHtml)
    .replace(/\{\{body\}\}/g, bodyHtml);
}
```

Note: `name` is HTML-escaped (plain text). `contact` uses `marked.parseInline` so markdown links work but no block-level elements sneak in. `body` is full markdown → HTML.

async function renderPdf(html, outputPath) {
  const browser = await chromium.launch();
  try {
    const page = await browser.newPage();
    // setContent with a baseURL so the <link rel="stylesheet" href="shared.css"> resolves
    await page.setContent(html, { waitUntil: "networkidle" });
    // Re-set with baseURL trick: Playwright's setContent doesn't accept baseURL,
    // so we navigate via data URL with the template dir served as files://.
    // Simpler: inline the CSS into the HTML before rendering.
    await page.pdf({
      path: outputPath,
      format: "Letter",
      printBackground: false,
    });
  } finally {
    await browser.close();
  }
}
```

**Decision point:** the `<link rel="stylesheet" href="shared.css">` in the templates won't resolve when HTML is passed to `setContent()` directly — there's no base URL. Two options:

- (a) Inline the CSS into a `<style>` tag before rendering
- (b) Navigate via `file://` URL to a temp HTML file in `templates/`

Option (a) is simpler and has no filesystem side effects. Update `renderHtml` to inline:

```javascript
async function main(argv) {
  const [, , inputArg, outputArg] = argv;
  if (!inputArg) {
    console.error("Usage: node scripts/generate-pdf.mjs <input.md> [output.pdf]");
    process.exit(2);
  }

  const inputPath = path.resolve(inputArg);
  if (!fs.existsSync(inputPath)) {
    console.error(`Input not found: ${inputPath}`);
    process.exit(1);
  }

  const outputPath = outputArg
    ? path.resolve(outputArg)
    : inputPath.replace(/\.md$/i, ".pdf");

  const templatePath = pickTemplate(inputPath);
  if (!fs.existsSync(templatePath)) {
    console.error(`Template not found: ${templatePath}`);
    process.exit(1);
  }

  const raw = fs.readFileSync(inputPath, "utf8");
  const { content: afterFrontmatter } = matter(raw);
  const normalized = normalizeUnicode(afterFrontmatter);

  const { name, contact, body } = parseResumeSections(normalized);
  const html = renderHtml({ name, contact, bodyMarkdown: body, templatePath });

  try {
    await renderPdf(html, outputPath);
  } catch (err) {
    const msg = String(err && err.message ? err.message : err);
    if (/Executable doesn't exist|browserType\.launch/.test(msg)) {
      console.error("Chromium not installed. Run: npx playwright install chromium");
    } else {
      console.error(`PDF rendering failed: ${msg}`);
    }
    process.exit(1);
  }

  const bytes = fs.statSync(outputPath).size;
  console.log(`Wrote ${outputPath} (${bytes} bytes)`);
}

// Only run main when executed as a script, not when imported for tests
const isMain = import.meta.url === `file://${process.argv[1].replace(/\\/g, "/")}`;
if (isMain) {
  main(process.argv).catch((err) => {
    console.error(err.stack || err.message || err);
    process.exit(1);
  });
}
```

**Note on the `isMain` check:** on Windows, `process.argv[1]` may use backslashes while `import.meta.url` uses forward slashes. The `.replace(/\\/g, "/")` normalizes them. This matters because the test file imports from `generate-pdf.mjs` — without this guard, running the test file would trigger the CLI.

## PDF generation

`resume-builder` and `resume-tailor` automatically produce ATS-safe PDFs alongside the markdown they save. Under the hood: `scripts/generate-pdf.mjs` renders the markdown through an HTML template with Playwright's headless Chromium.

### One-time setup

```bash
npm install
npx playwright install chromium
```

The second command downloads Chromium (~300 MB) into Playwright's cache. This is a one-time cost per machine.

### Manual invocation

Regenerate a PDF after hand-editing markdown:

```bash
node scripts/generate-pdf.mjs my-documents/resume.md
node scripts/generate-pdf.mjs my-documents/applications/acme-engineer/coverletter.md
```

Output goes to the same directory with a `.pdf` extension unless a second argument is passed.

### Templates

Two templates under `templates/` share one stylesheet (`shared.css`):

- `resume-template.html` — for resumes
- `coverletter-template.html` — for cover letters

The design is single-column, black-on-white serif typography. ATS parsers and human reviewers see the same document — the "ATS-safe vs. pretty" tradeoff is largely a myth when multi-column layouts, images, and text-in-shapes are avoided.

### Troubleshooting

If PDF generation fails, the markdown is still saved — it's the canonical artifact. The error message will tell you the fix; the most common case is Chromium not being installed (run the setup step above).
```

## Appendix: Why some things are NOT in this plan

- **No CI integration.** Repo has no CI; the manual smoke test + `scripts/test-generate-pdf.mjs` are the verification surface.
- **No template variants.** One visual design per Q2. Adding a second template (e.g., "senior exec") is a future plan, not this one.
- **No state layer changes.** PDF generation reads markdown and writes PDF. It does not touch `applications.md`, frontmatter, or reports. Confirmed in the design spec; double-check by not importing anything from `_shared/state-layer.md`.
- **No Canva MCP integration.** Explicitly out of scope per issue #2.
- **No `pdf: false` frontmatter toggle.** YAGNI — if iteration speed becomes a problem, add it then.
- **No mocking of Playwright in tests.** Mocking a browser is more work than running it and tells you less. The end-to-end smoke test IS the test.
